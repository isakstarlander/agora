# RFC-07 — Accountability API Endpoint

| Field  | Value |
|--------|-------|

---

## Summary

Implemented `GET /api/v1/accountability` — the cross-source compound endpoint that is
the primary reason this project exists. Given a party code and a topic, it returns the
complete accountability arc across all four data layers (promises, legislation, votes,
budget) plus a ≤150-word Swedish synthesis generated by Claude Sonnet. Also implemented
`GET /api/v1/parties/:party/alignment`, a public endpoint that returns aggregated
manifesto positions per Manifesto Project category. Both endpoints are registered in the
OpenAPI spec. `tsc --noEmit` exits clean.

---

## Motivation

Every prior RFC built infrastructure in service of this endpoint. The key system
(RFC-04), the search RPCs (RFC-06), and the manifesto embedding pipeline (RFC-03)
are all invoked here in a single request. The value proposition — showing how a party's
actual votes and budget allocations compare to its manifesto promises — requires joining
data from all four sources simultaneously. No single-source endpoint can do this.

---

## Decisions

### `classifyTopic` wraps the entire Anthropic call in `try/catch`

The spec's code sample wraps only the `JSON.parse` call. The outer `client.messages.create`
can also throw — notably with an `invalid_request_error` when the Anthropic account has
insufficient credits. This was observed during verification: the uncaught error propagated
through `apiRoute`'s generic catch handler and returned a 500 instead of gracefully
skipping the budget layer. The fix was to wrap the full `messages.create` call in the
`try/catch`, logging the error and returning `[]` (which causes the `budget` layer to
be skipped with an empty array rather than crashing the response).

### `@anthropic-ai/sdk` installed as a direct dependency of `apps/web`

The existing stack only had `@ai-sdk/anthropic` (the Vercel AI SDK wrapper). The
accountability route and `classify.ts` use `new Anthropic()` from `@anthropic-ai/sdk`
directly because the Vercel AI SDK abstraction does not expose the raw
`client.messages.create` interface needed for non-streaming structured calls with
explicit model and system content. The package was installed with `--engine-strict=false`
because the local environment runs Node 22 while the workspace engine field specifies
Node ≥24.

### `voyageai` loaded via `createRequire` in `embed.ts`

The `voyageai` package ships a broken ESM build — its `dist/esm/extended/index.mjs`
re-exports from relative directory paths (`../api`, `../errors`) that neither Turbopack
nor Node.js ESM can resolve. The ingestion scripts in `scripts/embed/generate-embeddings.ts`
already worked around this with `createRequire(import.meta.url)` to force the CJS
entrypoint. The same pattern was applied to `apps/web/lib/api/embed.ts`. Additionally,
`serverExternalPackages: ['voyageai']` was added to `next.config.ts` so Turbopack does
not attempt to bundle the module at all.

### `doc_type` and `doc_rm` passed as `undefined`, not `null`

Consistent with RFC-06: the generated Supabase types for the `search_documents` RPC
declare optional parameters as `string | undefined`. The spec samples pass `null`;
these were changed to `undefined` to satisfy the type checker.

### `party` params cast to `string` in dynamic segments

Consistent with RFC-05 and RFC-06: Next.js with `noUncheckedIndexedAccess` types
route segment params as `string | undefined`. All dynamic segment reads use an explicit
`as string` cast since the value is guaranteed present when the route matches.

### `ZodError.issues` instead of `.errors`

Consistent with prior RFCs: Zod v4 renamed `.errors` to `.issues`.

### Haiku model string confirmed valid

The spec uses `'claude-haiku-4-5-20251001'`. This string was confirmed as a valid
`ModelParam` literal in the `@anthropic-ai/sdk` TypeScript type definitions before
implementation.

### Alignment endpoint returns empty `categories` when statements lack `category_code`

`GET /api/v1/parties/MP/alignment` returned `manifesto_year: 2022` with an empty
`categories` array during verification. This is correct behaviour — the SQL filter
`.not('category_code', 'is', null)` excludes statements that were not tagged with a
Manifesto Project category code during ingestion. The endpoint code is correct; the data
completeness depends on the manifesto ingestion pipeline.

## Files Created

| File | Description |
|---|---|
| `apps/web/lib/api/classify.ts` | `classifyTopic(topic)` — calls Claude Haiku to map a free-text topic to 1–3 Swedish expenditure area codes; returns `[]` on any failure |
| `apps/web/lib/api/party-votes.ts` | `getPartyVotesOnDocuments(party, documentIds)` — tallies `vote_results` Ja/Nej for a party across a set of document IDs, returns aggregated `party_position` |
| `apps/web/app/api/v1/accountability/route.ts` | `GET /api/v1/accountability` — auth + rate limit + 4-layer parallel fetch + Claude Sonnet synthesis |
| `apps/web/app/api/v1/parties/[party]/alignment/route.ts` | `GET /api/v1/parties/:party/alignment` — public; latest manifesto categories with `mean_position` per Manifesto Project category |
